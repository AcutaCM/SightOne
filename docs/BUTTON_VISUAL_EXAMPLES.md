## Implementation Tips

### âœ… Good Examples

```tsx
// Clear hierarchy
<Button color="primary">Save</Button>
<Button variant="flat">Cancel</Button>

// Consistent pattern
<Button variant="solid">Submit</Button>
<Button variant="bordered">Preview</Button>
<Button variant="light">Skip</Button>

// Appropriate danger usage
<Button color="danger">Delete Account</Button>
<Button color="danger" variant="light">Remove</Button>
```

### âŒ Bad Examples

```tsx
// Too many primary buttons
<Button color="primary">Save</Button>
<Button color="primary">Cancel</Button>  // Should be secondary

// Inconsistent hierarchy
<Button variant="light">Submit</Button>  // Should be primary
<Button variant="solid">Skip</Button>    // Should be tertiary

// Custom styles breaking system
<Button className="bg-blue-500">Save</Button>  // Don't override
```

## Quick Reference

| Button Type | Opacity | Use Case | Variant |
|------------|---------|----------|---------|
| Primary | 100% | Main actions | `solid`, `color="primary"` |
| Secondary | 20% | Alternative actions | `flat`, `bordered` |
| Tertiary | 10% | Subtle actions | `light`, `ghost` |
| Danger | 90% | Destructive actions | `color="danger"` |
| Disabled | 3% | Inactive state | `disabled` |
| Loading | 60% | Processing state | `isLoading` |

---

**Remember**: The button transparency system creates visual hierarchy through opacity, not color. This maintains the elegant black-and-white aesthetic of the dark mode redesign.
