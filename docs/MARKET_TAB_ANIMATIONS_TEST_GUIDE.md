# 市场分栏动画测试指南

## 快速测试步骤

### 1. 启动应用

```bash
cd drone-analyzer-nextjs
npm run dev
```

### 2. 打开市场页面

1. 登录应用
2. 找到并点击打开市场（Marketplace）的按钮
3. 你应该能看到顶部的分栏导航：Home、Assistant、MCP Plugin、Model、Model Provider

### 3. 测试标签切换动画

#### 测试 1：基本切换
- **操作**：依次点击每个分栏标签
- **预期效果**：
  - 蓝色指示器平滑滑动到新标签下方
  - 指示器宽度自动调整匹配标签宽度
  - 激活标签的文字颜色变深，字重增加
  - 整个动画流畅，无卡顿

#### 测试 2：快速切换
- **操作**：快速连续点击多个不同的标签
- **预期效果**：
  - 指示器能够跟上点击速度
  - 动画不会重叠或错位
  - 最终停在最后点击的标签位置

#### 测试 3：悬停效果
- **操作**：将鼠标悬停在未激活的标签上
- **预期效果**：
  - 标签背景色变化（显示浅色背景）
  - 标签轻微向上移动（1px）
  - 文字颜色变深
  - 移开鼠标后平滑恢复

#### 测试 4：点击效果
- **操作**：按住鼠标在标签上
- **预期效果**：
  - 按下时标签回到原位置
  - 释放后切换到新标签

### 4. 测试响应式

#### 测试 5：窗口大小变化
- **操作**：调整浏览器窗口大小
- **预期效果**：
  - 指示器位置自动更新
  - 指示器始终对齐当前激活的标签

### 5. 测试主题切换

#### 测试 6：深色主题
- **操作**：切换到深色主题
- **预期效果**：
  - 指示器光晕效果更明显
  - 标签颜色适配深色背景
  - 悬停效果清晰可见

#### 测试 7：浅色主题
- **操作**：切换到浅色主题
- **预期效果**：
  - 指示器光晕效果适中
  - 标签颜色适配浅色背景
  - 整体视觉和谐

### 6. 性能测试

#### 测试 8：动画帧率
- **操作**：
  1. 打开浏览器开发工具（F12）
  2. 切换到 Performance 标签
  3. 开始录制
  4. 快速点击多个标签
  5. 停止录制并查看帧率

- **预期效果**：
  - 动画帧率应保持在 60fps 左右
  - 无明显的帧率下降

## 常见问题排查

### 问题 1：指示器位置不正确
- **可能原因**：标签元素尚未渲染完成
- **解决方案**：刷新页面，确保组件完全加载

### 问题 2：动画卡顿
- **可能原因**：浏览器性能问题或其他进程占用资源
- **解决方案**：
  - 关闭其他标签页
  - 检查 CPU 使用率
  - 尝试其他浏览器

### 问题 3：指示器不移动
- **可能原因**：JavaScript 错误
- **解决方案**：
  - 打开浏览器控制台查看错误信息
  - 检查 MarketTabComponents.tsx 是否正确导入

### 问题 4：样式不正确
- **可能原因**：CSS 变量未定义或主题未加载
- **解决方案**：
  - 检查 globals.css 是否正确加载
  - 确认 HeroUI 主题变量已定义
